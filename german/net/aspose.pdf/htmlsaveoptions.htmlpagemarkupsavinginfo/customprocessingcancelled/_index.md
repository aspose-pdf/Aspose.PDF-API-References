## HtmlSaveOptions.HtmlPageMarkupSavingInfo.CustomProcessingCancelled field

Sollte bei Bedarf im benutzerdefinierten Code gesetzt werden. Dieses Flag muss im benutzerdefinierten Code auf "true" gesetzt werden, wenn aus bestimmten Gründen bereitgestelltes HTML-Markup nicht mit benutzerdefiniertem Code, sondern mit dem Konvertercode selbst in Standard-für-Konverter-Weise verarbeitet werden soll. Das Setzen dieses Flags im benutzerdefinierten Code bedeutet also, dass der benutzerdefinierte Code die referenzierte Datei nicht verarbeitet hat und der Konverter selbst damit umgehen muss
