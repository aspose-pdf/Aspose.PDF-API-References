## HtmlSaveOptions.PartsEmbeddingModes enumeration

Diese Aufzählung listet mögliche Modi zum Einbetten von Dateien auf, auf die in HTML verwiesen wird Es ermöglicht zu steuern, ob referenzierte Dateien (HTML, Schriftarten, Bilder, CSS) in die Haupt-HTML-Datei eingebettet oder als getrennte Binäreinheiten generiert werden

```csharp
public enum PartsEmbeddingModes
```

### Werte

| Name | Wert | Beschreibung |
| --- | --- | --- |
| EmbedAllIntoHtml | `0` | Erzwingt das Einbetten aller referenzierten Dateien (Css, Bilder, Schriftarten) in generiertes HTML-Markup (dh in HTML selbst) Dieser Ansatz generiert eine HTML-Datei, aber die Gesamtgröße der Ausgabe wird größer (weil Base64-Codierung von Binärdateien verwendet wird) und nicht alle Browser (insbesondere ältere) verarbeiten erfolgreich Binärdateien, die in HTML eingebettet sind. Aber es ermöglicht das Abrufen von HTML, das das gesamte Ergebnis enthält, ohne zusätzliche Dateien. |
| EmbedCssOnly | `1` | Erzwingt das Trennen aller referenzierten Dateien außer CSS (Bilder und Schriftarten) D. h. CSS wird in das Ergebnis-HTML eingebettet, und alle anderen referenzierten Dateien (Bilder und Schriftarten) werden als externe Teile verarbeitet Es generiert HTML, das für einen breiten Satz geeignet ist von browsers |
| NoEmbedding | `2` | Erzwingt das Auseinanderlegen referenzierter Dateien (Css, Bilder, Schriftarten) Dieser Ansatz generiert eine Reihe von Dateien, aber die Gesamtgröße der Ausgabe wird kleiner (weil keine Base64-Codierung von Binärdateien verwendet wird) Auch ein solcher Ansatz generiert HTML, für das es geeignet ist große Auswahl an Browsern |
