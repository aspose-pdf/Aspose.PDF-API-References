## Enumerazione CollectionFieldSubtype

Rappresenta il parametro di sottotipo di un campo in una collezione di schemi.

```csharp
public enum CollectionFieldSubtype
```

### Valori

| Nome | Valore | Descrizione |
| --- | --- | --- |
| None | `0` | Il sottotipo non è definito. |
| S | `1` | Un tipo di testo. I dati del campo devono essere memorizzati come una stringa di testo PDF. |
| D | `2` | Un tipo di data. I dati del campo devono essere memorizzati come una stringa di data PDF. |
| N | `3` | Un tipo di numero. I dati del campo devono essere memorizzati come un numero PDF. |
| F | `4` | I dati del campo devono essere il nome del file del flusso di file incorporato, come identificato dall'entrata UF della specifica del file, se presente; altrimenti dall'entrata F della specifica del file |
| Desc | `5` | I dati del campo devono essere il nome del file del flusso di file incorporato, come identificato dall'entrata UF della specifica del file, se presente; altrimenti dall'entrata F della specifica del file |
| ModDate | `6` | I dati del campo devono essere la data di modifica del flusso di file incorporato, come identificato dall'entrata ModDate nel dizionario dei parametri del file incorporato. |
| CreationDate | `7` | I dati del campo devono essere la data di creazione del flusso di file incorporato, come identificato dall'entrata CreationDate nel file incorporato |
| Size | `8` | I dati del campo devono essere la dimensione del file incorporato, come identificato dall'entrata Size nel dizionario dei parametri del file incorporato |
| CompressedSize | `9` | (PDF 2.0) I dati del campo sono la lunghezza del flusso di file incorporato, come identificato dall'entrata Length nel dizionario del flusso di file incorporato, e i due valori devono essere identici. |
