## HtmlSaveOptions.PartsEmbeddingModes enumeration

Questo enum enumera le possibili modalità di incorporamento dei file referenziati in HTML Consente di controllare se i file referenziati (HTML, Font, Immagini, CSS) verranno incorporati nel file HTML principale o verranno generati come entità binarie a parte

```csharp
public enum PartsEmbeddingModes
```

### I valori

| Nome | Valore | Descrizione |
| --- | --- | --- |
| EmbedAllIntoHtml | `0` | Impone l'incorporamento di tutti i file di riferimento (Css, Immagini, Font) nel markup HTML generato (cioè nello stesso HTML) Questo approccio genera un file HTML, ma la dimensione totale dell'output aumenta (perché è in uso la codifica Base64 dei binari) e non tutti i browser (soprattutto quelli legacy) elaborano correttamente i binari incorporati in HTML. Ma consente di ottenere HTML che contiene l'intero risultato, senza file aggiuntivi. |
| EmbedCssOnly | `1` | Impone mettere a parte tutti i file di riferimento tranne CSS (immagini e caratteri) Cioè CSS verrà incorporato nel risultato HTML e tutti gli altri file di riferimento (immagini e caratteri) verranno elaborati come parti esterne Genera HTML che è adatto per un ampio set di browser |
| NoEmbedding | `2` | Impone la messa a parte dei file di riferimento (Css, Immagini, Font) Questo approccio genera un insieme di file, ma la dimensione totale dell'output diventa più piccola (perché non è in uso la codifica Base64 dei binari) Anche tale approccio genera HTML adatto per ampio set di browser |
