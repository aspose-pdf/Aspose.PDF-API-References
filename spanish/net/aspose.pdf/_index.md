| [PdfASymbolicFontEncodingStrategy](./pdfasymbolicfontencodingstrategy/) | Esta clase describe reglas que se pueden utilizar para ajustar el proceso de copia de datos de codificación para casos en los que la fuente simbólica TrueType tiene más de una codificación. Algunos documentos PDF después de la conversión al formato PDF/A podrían dar un error "Más de una codificación en la cmap de la fuente simbólica TrueType". ¿Cuál es la razón de este error? Todas las fuentes simbólicas TrueType tienen una tabla especial "cmap" en sus datos internos. Esta tabla mapea códigos de caracteres a índices de glifos. Y esta tabla podría contener diferentes subtables de codificación que describen las codificaciones utilizadas. Ver información avanzada sobre tablas cmap en https://developer.apple.com/fonts/TrueType-Reference-Manual/RM06/Chap6cmap.html. Normalmente, la tabla cmap contiene varias subtables de codificación, pero el estándar PDF/A requiere que solo se deje una subtabla de codificación para esta fuente en el documento PDF/A o que debe haber una subtabla de codificación (3,0) entre estas subtables de fuente. Y la pregunta clave aquí es: ¿qué datos deben tomarse de otras subtables para copiar en la tabla de codificación de destino (3,0)? La mayoría de las fuentes tienen tablas cmap 'bien formadas' donde cada subtabla de codificación es completamente consistente con otra subtabla. Pero algunas fuentes tienen tablas cmap con colisiones, donde, por ejemplo, una subtabla tiene el índice de glifo 100 para unicode 100, pero otra subtabla tiene el índice de glifo 200 para el mismo unicode 100. Para resolver estos problemas se necesita una estrategia especial. Por defecto, se utiliza la siguiente estrategia: se busca la subtabla mac (1,0). Si se encuentra esta tabla, solo se utilizan estos datos para llenar la tabla de destino (3,0). Si no se encuentra la subtabla mac, entonces se iteran todas las subtables excepto (3,0) y se utilizan para copiar datos en la subtabla de destino (3,0). Además, el mapeo para cada unicode (unicode, índice de glifo) se copia en la tabla de destino solo si la tabla de destino no tiene este unicode en ese momento. Así que, por ejemplo, si la primera subtabla tiene el índice de glifo 100 para unicode 100, y la siguiente subtabla tiene el índice de glifo 200 para el mismo unicode 100, solo se copiarán los datos de la primera subtabla (unicode=100, índice de glifo = 100). Así que cada subtabla anterior tiene prioridad sobre la siguiente. Las propiedades de esta clase [`PdfASymbolicFontEncodingStrategy`](../aspose.pdf/pdfasymbolicfontencodingstrategy/) ayudan a ajustar el comportamiento predeterminado. Si la propiedad [`PreferredCmapEncodingTable`](../aspose.pdf/pdfasymbolicfontencodingstrategy/preferredcmapencodingtable/) de tipo [`CMapEncodingTableType`](../aspose.pdf/pdfasymbolicfontencodingstrategy.queueitem.cmapencodingtabletype/) está configurada, entonces la subtabla relevante se utilizará en prioridad sobre la subtabla mac (1,0). El valor 'MacTable' de la enumeración [`CMapEncodingTableType`](../aspose.pdf/pdfasymbolicfontencodingstrategy.queueitem.cmapencodingtabletype/) no tiene sentido en este caso, porque apunta a la misma subtabla mac (1,0) que se utilizará por defecto. La propiedad [`CmapEncodingTablesPriorityQueue`](../aspose.pdf/pdfasymbolicfontencodingstrategy/cmapencodingtablespriorityqueue/) descarta todas las prioridades para cualquier subtabla. Si esta propiedad está configurada, entonces solo se utilizarán las subtables de la cola declarada en el orden especificado. Si las subtables especificadas no se encuentran, entonces se utilizará la iteración predeterminada de todas las subtables y la estrategia de copia descrita anteriormente. El objeto [`QueueItem`](../aspose.pdf/pdfasymbolicfontencodingstrategy.queueitem/) especifica la subtabla de codificación utilizada. Esta subtabla se puede establecer mediante una combinación de miembros (PlatformID, PlatformSpecificId) o mediante la enumeración [`CMapEncodingTableType`](../aspose.pdf/pdfasymbolicfontencodingstrategy.queueitem.cmapencodingtabletype/). En caso de que la fuente no tenga la subtabla (3,0), se utilizará alguna otra subtabla para mantener la compatibilidad con PDF/A. La elección de la subtabla a utilizar se realiza bajo las mismas reglas que se describieron anteriormente, de modo que las propiedades [`PreferredCmapEncodingTable`](../aspose.pdf/pdfasymbolicfontencodingstrategy/preferredcmapencodingtable/) y [`CmapEncodingTablesPriorityQueue`](../aspose.pdf/pdfasymbolicfontencodingstrategy/cmapencodingtablespriorityqueue/) se utilizan para determinar la subtabla resultante, y si la fuente no tiene las subtablas solicitadas, se utilizará cualquier subtabla existente. |
