## HtmlSaveOptions.HtmlPageMarkupSavingInfo class

Si la propiedad SplitToPages de HtmlSaveOptions, se crean varios archivos HTML (un archivo HTML por página convertida) durante la conversión de PDF a HTML. Esta clase representa un conjunto de datos relacionados con el guardado personalizado del marcado de una página HTML durante la conversión de PDF a HTML

| [CustomProcessingCancelled](../../aspose.pdf/htmlpagemarkupsavinginfo/customprocessingcancelled) | Debe configurarse en código personalizado cuando sea necesario. Este indicador debe configurarse en "verdadero" en código personalizado si, por alguna razón, el marcado html proporcionado por debe procesarse no con código personalizado sino con el propio código del convertidor de forma estándar para el convertidor. Por lo tanto, configurar si este indicador en el código personalizado significa que el código personalizado no procesó el archivo al que se hace referencia y el convertidor debe manejarlo por sí mismo |
| [HtmlHostPageNumber](../../aspose.pdf/htmlpagemarkupsavinginfo/htmlhostpagenumber) | Establecido por convertidor. Si se establece la propiedad SplitToPages, se crean varios archivos HTML (un archivo HTML por página convertida) durante la conversión. Esta propiedad contiene el ordinal del archivo de la página HTML guardada. La propiedad se puede usar en la lógica del código personalizado para decidir cómo procesar o dónde guardar la página HTML y Si la división en páginas está desactivada, este valor siempre contiene '1' ya que en tal caso, solo se genera una gran página HTML para todo el documento de origen. |
| [PdfHostPageNumber](../../aspose.pdf/htmlpagemarkupsavinginfo/pdfhostpagenumber) | Establecido por el convertidor. Si se establece la propiedad SplitToPages, se crean varios archivos HTML (un archivo HTML por página convertida) durante la conversión creada . Esta propiedad le dice al código personalizado desde qué página del PDF original se creó el marcado HTML guardado . Si el número de página original por alguna razón es desconocido o SplitOnPages=false, entonces esta propiedad siempre contiene '0' que indica que el convertidor no puede proporcionar el número de página del PDF original exacto para el archivo de marcado HTML provisto. |
