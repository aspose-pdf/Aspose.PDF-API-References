## XslFoLoadOptions.ParsingErrorsHandlingTypes enumeration

El documento XSLFO de origen puede contener errores de formato. Esta enumeración enumera posibles estrategias de manejo de tales errores de formato

```csharp
public enum ParsingErrorsHandlingTypes
```

### Valores

| Nombre | Valor | Descripción |
| --- | --- | --- |
| TryIgnore | `0` | En este caso, se le indicará al convertidor que intente continuar con la conversión e ignore los errores de formato encontrados. En este caso, el éxito no está garantizado, pueden ocurrir problemas graves más adelante en el convertidor, y en tal caso, se generará una excepción con la lista de formatos encontrados errores. |
| ThrowExceptionImmediately | `1` | En este caso, la conversión se detendrá inmediatamente y se generará una excepción inmediatamente después de detectar del primer error de formato |
| InvokeCustomHandler | `2` | Este es el método más ágil: el código personalizado debe proporcionar (en la propiedad WarningCallback) un controlador especial que se llamará cuando se detecte un error de formato. Ese controlador puede registrar o contar errores, etc. y proporcionará la decisión de si el procesamiento puede continuar para este o aquel error. |
