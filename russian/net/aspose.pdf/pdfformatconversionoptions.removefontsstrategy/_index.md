## Перечисление PdfFormatConversionOptions.RemoveFontsStrategy

Некоторые документы имеют большой размер после конвертации в формат PDF/A. Чтобы уменьшить размер файла для этих документов, необходимо определить стратегию удаления шрифтов. Эта перечисление объявляет стратегии, которые могут быть использованы для оптимизации использования шрифтов. Каждая стратегия из этого перечисления имеет смысл только тогда, когда установлен флаг [`OptimizeFileSize`](../pdfformatconversionoptions/optimizefilesize/).

```csharp
[Flags]
public enum RemoveFontsStrategy : byte
```

### Значения

| Имя | Значение | Описание |
| --- | --- | --- |
| RemoveDuplicatedFonts | `4` | Эта стратегия удаляет все шрифты, которые имеют дубликаты в документе. Если документ содержит группу дублированных шрифтов, только один шрифт из этой группы встраивается в документ. Все остальные шрифты из этой группы удаляются из документа, каждый удаленный шрифт заменяется уже встроенным аналогом. |
| RemoveSimilarFontsWithDifferentWidths | `1` | Эта стратегия похожа на RemoveDuplicatedFonts, но она удаляет не полностью дублированные шрифты, а шрифты, которые похожи друг на друга и отличаются только параметром "Widths". Этот параметр содержит набор некоторых ширин для указанных символов шрифта. Каждое значение ширины из этого набора "Widths" не является реальной шириной символа (глифа), реальная ширина для этого символа уже определена в бинарных данных шрифта. Значение ширины из набора "Widths" означает визуальную ширину для этого символа - ширину, которую программное обеспечение для просмотра PDF должно установить при отображении символа вместо реальной ширины, определенной в шрифте. Более точно спецификация говорит: просмотрщики Acrobat 5.0 и более поздних версий используют ширины глифов, хранящиеся в словаре шрифтов, чтобы переопределить ширины глифов в самой программе шрифта, что улучшает согласованность отображения и печати документа. Эта стратегия более эффективна, чем RemoveDuplicatedFonts, но использование этой стратегии в некоторых случаях теоретически может повредить визуальному представлению конвертированного документа. Этот дефект возможен из-за того, что объявленные ширины шрифтов могут отличаться для одного и того же символа, и в этом случае ширина этого символа будет изменена на новую после замены шрифта - когда удаленный шрифт будет заменен в документе уже встроенным. И если визуальная ширина символа изменится - он будет отображаться неправильно, и это различие может вызвать визуальные дефекты, такие как наложение текста или другие проблемы. Но описанный визуальный дефект является очень редким случаем, и эта стратегия более эффективно уменьшает размер документа. |
| SubsetFonts | `2` | Это наиболее эффективная стратегия для уменьшения размера документа. Она берет полностью встроенные наборы шрифтов и сокращает их до только тех подмножеств, которые используются. Рекомендуется использовать эту стратегию в сочетании с RemoveDuplicatedFonts или RemoveSimilarFontsWithDifferentWidths, чтобы получить множественный эффект сжатия для размера файла. Использование всех трех стратегий одновременно не имеет смысла, и стратегия RemoveSimilarFontsWithDifferentWidths не будет использоваться в этом случае. |
