## CollectionSort class


Представляет класс для определения сортировки коллекции.

```cpp
class CollectionSort : public System::Object
```

## Методы

| Метод | Описание |
| --- | --- |
| [CollectionSort](./collectionsort/)() | Создаёт экземпляр класса сортировки коллекции. |
| [CollectionSort](./collectionsort/)(const System::SharedPtr\<Engine::Data::IPdfDictionary\>\&) | Создаёт экземпляр класса сортировки коллекции. |
| [get_A](./get_a/)() const | Получает значение порядка сортировки. Если значение является логическим, оно указывает, будет ли интерактивный процессор PDF сортировать элементы в коллекции в порядке возрастания (true) или убывания (false). Если значение является массивом, каждый элемент массива должен быть логическим значением, которое указывает, будет ли запись с тем же индексом в массиве S сортироваться в порядке возрастания или убывания. Если количество записей в массиве A больше, чем количество записей в массиве S, лишние записи в массиве A игнорируются. Если количество записей в массиве A меньше, чем количество записей в массиве S, отсутствующие записи в массиве A считаются истинными. Значение по умолчанию: **true** |
| [get_S](./get_s/)() const | Получает имя или имена полей, которые интерактивный процессор PDF будет использовать для сортировки элементов в коллекции. Если значение является именем, оно идентифицирует поле, описанное в словаре родительской коллекции. Если значение является массивом, каждый элемент массива должен быть именем, которое идентифицирует поле, описанное в словаре родительской коллекции. Формат массива используется, чтобы позволить дополнительным полям участвовать в сортировке, где каждое дополнительное поле используется для разбивки связей. Более конкретно, если несколько словарей элементов коллекции имеют одинаковое значение для первого поля, указанного в массиве, значения последующих полей, указанных в массиве, будут использоваться для сортировки, пока не будет определён уникальный порядок или пока не исчерпаются указанные поля. |
