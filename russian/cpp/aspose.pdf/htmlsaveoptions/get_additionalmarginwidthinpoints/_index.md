## HtmlSaveOptions::get_AdditionalMarginWidthInPoints method


Если атрибут 'SplitOnPages=false', тогда весь HTML, представляющий все входные страницы PDF, не будет разбит на отдельные HTML‑страницы, а будет помещён в один большой результирующий HTML‑файл. Однако каждая исходная страница PDF будет представлена своей прямоугольной областью в HTML (при необходимости эти области могут быть обведены, чтобы показать края листа с помощью специального атрибута 'PageBorderIfAny'). Этот параметр определяет ширину отступа, который будет принудительно оставлен вокруг этих выходных HTML‑областей, представляющих страницы исходного PDF‑документа. По сути он задаёт гарантированный интервал между HTML‑представлениями "paper" страниц PDF в таком режиме конвертации.

```cpp
int32_t Aspose::Pdf::HtmlSaveOptions::get_AdditionalMarginWidthInPoints() const
```


## Deprecated
AdditionalMarginWidthInPoints устарел, пожалуйста, используйте PageMarginIfAny вместо этого.
