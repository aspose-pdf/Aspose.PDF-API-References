## Classe PdfASymbolicFontEncodingStrategy

Cette classe décrit les règles pouvant être utilisées pour ajuster le processus de copie des données d'encodage dans les cas où une police symbolique TrueType possède plus d'un encodage. Certains documents PDF, après conversion au format PDF/A, peuvent générer une erreur « More than one encoding in symbolic TrueType font's cmap ». Quelle est la cause de cette erreur ? Toutes les polices symboliques TrueType possèdent une table spéciale « cmap » dans leurs données internes. Cette table associe des codes de caractères à des indices de glyphes. Et cette table peut contenir différentes sous-tables d'encodage qui décrivent les encodages utilisés. Voir les informations avancées sur les tables cmap à https://developer.apple.com/fonts/TrueType-Reference-Manual/RM06/Chap6cmap.html. Habituellement, la table cmap contient plusieurs sous-tables d'encodage, mais la norme PDF/A exige qu'il ne reste qu'une seule sous-table d'encodage pour cette police dans le document PDF/A ou qu'une sous-table d'encodage (3,0) figure parmi les sous-tables de cette police. Et la question clé ici est : quelles données doivent être récupérées dans les autres sous-tables pour être copiées dans la table d'encodage de destination (3,0) ? La majorité des polices possèdent des tables cmap « bien formées » où chaque sous-table d'encodage est entièrement cohérente par rapport aux autres. Mais certaines polices ont des tables cmap avec des collisions – par exemple, une sous-table peut avoir l'indice de glyphe 100 pour l'unicode 100, tandis qu'une autre sous-table a l'indice de glyphe 200 pour le même unicode 100. Pour résoudre ces problèmes, une stratégie spéciale est nécessaire. Par défaut, la stratégie suivante est utilisée : on recherche la sous-table mac(1,0). Si cette table est trouvée, seules ces données sont utilisées pour remplir la table de destination (3,0). Si la sous-table mac n'est pas trouvée, alors toutes les sous-tables, sauf (3,0), sont parcourues et utilisées pour copier les données dans la sous-table de destination (3,0). De plus, la correspondance pour chaque paire (unicode, indice de glyphe) est copiée dans la table de destination uniquement si celle-ci ne contient pas déjà cet unicode à ce moment-là. Ainsi, par exemple, si la première sous-table a l'indice de glyphe 100 pour l'unicode 100, et que la sous-table suivante a l'indice de glyphe 200 pour le même unicode 100, seules les données de la première sous-table (unicode=100, indice de glyphe = 100) seront copiées. Ainsi, chaque sous-table précédente a la priorité sur la suivante. Les propriétés de cette classe `PdfASymbolicFontEncodingStrategy` aident à ajuster le comportement par défaut. Si la propriété [`PreferredCmapEncodingTable`](./preferredcmapencodingtable/) de type [`CMapEncodingTableType`](../pdfasymbolicfontencodingstrategy.queueitem.cmapencodingtabletype/) est définie, alors la sous-table correspondante sera utilisée en priorité par rapport à la sous-table mac(1,0). La valeur 'MacTable' de l'énumération [`CMapEncodingTableType`](../pdfasymbolicfontencodingstrategy.queueitem.cmapencodingtabletype/) n'a aucun sens dans ce cas, car elle renvoie à la même sous-table mac(1,0) qui sera utilisée par défaut. La propriété [`CmapEncodingTablesPriorityQueue`](./cmapencodingtablespriorityqueue/) annule toutes les priorités pour toute sous-table. Si cette propriété est définie, alors seules les sous-tables de la file d'attente déclarée seront utilisées dans l'ordre spécifié. Si les sous-tables spécifiées ne sont pas trouvées, alors l'itération par défaut de toutes les sous-tables et la stratégie de copie décrite ci-dessus seront utilisées. L'objet [`QueueItem`](../pdfasymbolicfontencodingstrategy.queueitem/) spécifie la sous-table d'encodage utilisée. Cette sous-table peut être définie via une combinaison des membres (PlatformID, PlatformSpecificId) ou via l'énumération [`CMapEncodingTableType`](../pdfasymbolicfontencodingstrategy.queueitem.cmapencodingtabletype/). Dans le cas où la police ne possède pas la sous-table (3,0), une autre sous-table sera utilisée pour maintenir la compatibilité PDF/A. Le choix de la sous-table à utiliser s'effectue selon les mêmes règles que celles décrites précédemment, de sorte que les propriétés [`PreferredCmapEncodingTable`](./preferredcmapencodingtable/) et [`CmapEncodingTablesPriorityQueue`](./cmapencodingtablespriorityqueue/) sont utilisées pour déterminer la sous-table résultante, et si la police ne possède pas non plus les sous-tables demandées, alors toute sous-table existante sera utilisée.
