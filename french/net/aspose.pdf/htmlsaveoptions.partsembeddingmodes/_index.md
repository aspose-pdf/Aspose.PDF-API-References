## HtmlSaveOptions.PartsEmbeddingModes enumeration

Cette énumération énumère les modes possibles d'intégration des fichiers référencés dans HTML Elle permet de contrôler si les fichiers référencés (HTML, Polices, Images, CSS) seront intégrés dans le fichier HTML principal ou seront générés en tant qu'entités binaires séparées

```csharp
public enum PartsEmbeddingModes
```

### Valeurs

| Nom | Évaluer | La description |
| --- | --- | --- |
| EmbedAllIntoHtml | `0` | Applique l'intégration de tous les fichiers référencés (Css, Images, Polices) dans le balisage HTML généré (c'est-à-dire dans le HTML lui-même) Cette approche génère un fichier HTML, mais la taille totale de la sortie devient plus grande (car l'encodage Base64 des binaires est utilisé) et tous les navigateurs (en particulier les anciens) ne traitent pas avec succès les fichiers binaires intégrés au HTML. Mais cela permet d'obtenir du HTML contenant le résultat complet, sans aucun fichier supplémentaire. |
| EmbedCssOnly | `1` | Applique la séparation de tous les fichiers référencés à l'exception de CSS (Images et polices) C'est-à-dire que CSS sera intégré dans le résultat HTML et tous les autres fichiers référencés (Images et polices) seront traités comme des parties externes Il génère du HTML adapté à un large ensemble de navigateurs |
| NoEmbedding | `2` | Applique les fichiers référencés mis à part (Css, Images, Polices) Cette approche génère un ensemble de fichiers, mais la taille totale de la sortie devient plus petite (car aucun encodage Base64 des binaires n'est utilisé) Une telle approche génère également du HTML qui convient à large éventail de navigateurs |
