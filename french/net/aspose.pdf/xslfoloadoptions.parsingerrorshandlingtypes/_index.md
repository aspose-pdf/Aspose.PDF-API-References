## XslFoLoadOptions.ParsingErrorsHandlingTypes enumeration

Le document XSLFO source peut contenir des erreurs de formatage. Cette énumération énumère les stratégies possibles de gestion de telles erreurs de formatage

```csharp
public enum ParsingErrorsHandlingTypes
```

### Valeurs

| Nom | Évaluer | La description |
| --- | --- | --- |
| TryIgnore | `0` | Dans ce cas, le convertisseur sera invité à essayer de procéder avec la conversion et à ignorer les erreurs de formatage trouvées. Dans ce cas, le succès n'est pas garanti, de graves problèmes peuvent survenir plus tard dans le convertisseur, et en cas d'échec, une exception sera émise avec la liste des formatages trouvés erreurs. |
| ThrowExceptionImmediately | `1` | Dans ce cas, la conversion sera arrêtée immédiatement et une exception sera lancée immédiatement après la détection de la première erreur de formatage |
| InvokeCustomHandler | `2` | Il s'agit de la méthode la plus agile - le code personnalisé doit fournir (dans la propriété WarningCallback) le gestionnaire special qui sera appelé lorsqu'une erreur de formatage est détectée. Ce gestionnaire peut enregistrer ou compter les erreurs, etc. et fournira une décision si le traitement peut être poursuivi pour telle ou telle erreur. |
