## Champ HtmlSaveOptions.PagesFlowTypeDependsOnViewersScreenSize

Si l'attribut 'SplitOnPages=false', alors tout le HTML représentant toutes les pages PDF d'entrée sera mis dans un grand fichier HTML résultant. Ce drapeau définit si le HTML résultant sera généré de telle manière que le flux des zones représentant les pages PDF dans le HTML résultant dépendra de la résolution d'écran du visualiseur. Supposons que la largeur de l'écran du côté du visualiseur soit suffisamment grande pour mettre 2 pages ou plus côte à côte dans la direction horizontale. Si ce drapeau est défini sur true, alors cette opportunité sera utilisée (autant de pages que possible seront affichées dans la direction horizontale côte à côte, puis le prochain groupe horizontal de pages sera affiché sous le premier). Sinon, les pages s'écouleront de telle manière que la prochaine page ira toujours sous la précédente.

```csharp
public bool PagesFlowTypeDependsOnViewersScreenSize;
```
