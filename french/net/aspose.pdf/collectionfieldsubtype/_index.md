## Énumération CollectionFieldSubtype

Représente le paramètre de sous-type d'un champ dans une collection de schémas.

```csharp
public enum CollectionFieldSubtype
```

### Valeurs

| Nom | Valeur | Description |
| --- | --- | --- |
| Aucun | `0` | Le sous-type n'est pas défini. |
| S | `1` | Un type texte. Les données du champ doivent être stockées sous forme de chaîne de texte PDF. |
| D | `2` | Un type date. Les données du champ doivent être stockées sous forme de chaîne de date PDF. |
| N | `3` | Un type nombre. Les données du champ doivent être stockées sous forme de nombre PDF. |
| F | `4` | Les données du champ doivent être le nom de fichier du flux de fichier intégré, tel qu'identifié par l'entrée UF de la spécification de fichier, si présente ; sinon par l'entrée F de la spécification de fichier |
| Desc | `5` | Les données du champ doivent être le nom de fichier du flux de fichier intégré, tel qu'identifié par l'entrée UF de la spécification de fichier, si présente ; sinon par l'entrée F de la spécification de fichier |
| ModDate | `6` | Les données du champ doivent être la date de modification du flux de fichier intégré, tel qu'identifié par l'entrée ModDate dans le dictionnaire de paramètres de fichier intégré. |
| CreationDate | `7` | Les données du champ doivent être la date de création du flux de fichier intégré, tel qu'identifié par l'entrée CreationDate dans le fichier intégré |
| Size | `8` | Les données du champ doivent être la taille du fichier intégré, tel qu'identifié par l'entrée Size dans le dictionnaire de paramètres de fichier intégré |
| CompressedSize | `9` | (PDF 2.0) Les données du champ sont la longueur du flux de fichier intégré, tel qu'identifié par l'entrée Length dans le dictionnaire de flux de fichier intégré, et les deux valeurs doivent être identiques. |
