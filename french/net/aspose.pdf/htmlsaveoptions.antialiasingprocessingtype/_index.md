## HtmlSaveOptions.AntialiasingProcessingType enumeration

Cette énumération décrit les mesures d'anticrénelage possibles lors de la conversion

```csharp
public enum AntialiasingProcessingType
```

### Valeurs

| Nom | Évaluer | La description |
| --- | --- | --- |
| NoAdditionalProcessing | `0` | aucun traitement d'anticrénelage spécial n'est utilisé. Il s'agit d'une option optimale pour la majorité écrasante des documents et elle ne nécessite pas de temps supplémentaire lors de la conversion |
| TryCorrectResultHtml | `1` | Dans ce cas, le convertisseur essaie de détecter les endroits avec des éléments graphiques d'arrière-plan adjacents et de corriger le résultat HTML de manière pertinente. Cette option permet d'améliorer le résultat de l'exportation pour les documents contenant des arrière-plans construits à partir de plusieurs éléments graphiques adjacents (pour ce type de documents, les moteurs de rendu PDF , par exemple Acrobat Reader, essaie généralement de lisser les limites des éléments lors du rendu. Avec cette option, le convertisseur imite le comportement des moteurs de rendu PDF. Cette option permet d'améliorer la mise en page du résultat de l'exportation pour certains documents spécifiques (qui utilisent de tels arrière-plans composés), mais cela nécessite un temps supplémentaire pour le traitement (généralement environ 10 à 15 % de temps supplémentaire). L'utilisation de ce mode dans un cas général n'est donc pas recommandée. |
