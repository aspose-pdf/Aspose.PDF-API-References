## EpubSaveOptions.RecognitionMode enumeration

在转换PDF文件（通常具有固定布局）时， 转换引擎尝试进行分组和多级分析以恢复 原始文档作者的意图和产生的结果流布局。 这个属性调整转换为这个或那个 理想的内容识别方法。

```csharp
public enum RecognitionMode
```

### 价值观

| 姓名 | 价值 | 描述 |
| --- | --- | --- |
| Flow | `0` | 全识别模式，引擎尝试进行分组和多层次分析，还原 原文档作者的意图，并在流式布局中生成 xhtml . |
| PdfFlow | `1` | 这种转换的主要思想是基于保存在处理 pdf 文档期间形成的内容呈现的“自然”顺序。 在一般情况下，pdf文档保持自上而下\从左到右的呈现顺序（见附件directions.png）。这个假设允许 创建一个单路径算法，它将具有位置（固定布局）的 Aps 元素转换为 HTML、EPUB、DOC 等流格式。 这种模式对于从 PDF(APS) 转换为 EPUB 尤其有用，因为 EPUB 格式是为 Kindle 或 智能手机等电子阅读器开发的。这些设备的屏幕尺寸通常小于普通PC的屏幕尺寸。因此EPUB文档的内容最好保存为流格式 ，以便在不同大小的屏幕上正确呈现。 在这种模式下，每一列都将添加到前一列的末尾，这允许在 EPUB 阅读器的“分页”期间保持转换后文档 的逻辑结构。此成就允许正确呈现科学或杂志文章。 |
| Fixed | `2` | 这种模式快速且有利于最大限度地保留原始外观页面 但不幸的是，许多 EPUB 阅读器不支持固定布局的 xhtml |
