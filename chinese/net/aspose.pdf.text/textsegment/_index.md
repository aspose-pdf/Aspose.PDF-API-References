### 评论

简而言之，[`TextSegment`](../textsegment)对象是[`TextFragment`](../textfragment)对象的子对象。 详细信息: Pdf中 pdf 文档的文本是由两个基本对象表示:[`TextFragment`](../textfragment)和[`TextSegment`](../textsegment) 它们之间的差异主要取决于上下文。 让我们考虑以下场景。用户搜索文本“hello world”以对其进行操作、更改其属性、外观等 &lt;code&gt; Document doc = new Document(docFile); TextFragmentAbsorber 吸收器 = new TextFragmentAbsorber("hello world"); doc.Pages[1].Accept(absorber); &lt;/code&gt; 物理上 pdf 文本的表示非常复杂。 文本“hello world”可能由几个物理上独立的文本段组成。 Aspose.Pdf 文本模型基本建立了[`TextFragment`](../textfragment)object 提供单一的在物理[`TextSegment`](../textsegment)表示用户查询的对象集上设置的逻辑操作。 在文本搜索场景中，[`TextFragment`](../textfragment)是逻辑的“hello world”文本表示， 和[`TextSegment`](../textsegment)对象集合表示构造“hello world”文本对象的所有物理段。 所以，[`TextFragment`](../textfragment)接近于逻辑文本表示。 和[`TextSegment`](../textsegment)接近于物理文本表示。 显然每个[`TextSegment`](../textsegment)对象可能有它自己的字体、颜色、定位属性。 [`TextFragment`](../textfragment)提供简单的方法来更改文本及其属性:设置字体、设置字体大小、设置字体颜色等 同时[`TextSegment`](../textsegment)对象可以访问并且用户可以使用TextSegment对象独立。

### 例子

该示例演示了如何使用更改文本的文本颜色和字体大小[`TextState`](./textstate)对象的[`TextSegment`](../textsegment)对象。

```csharp
// 打开文档
cument doc = new Document(@"D:\Tests\input.pdf");

// 创建 TextFragmentAbsorber 对象以查找所有“hello world”文本出现
xtFragmentAbsorber absorber = new TextFragmentAbsorber("hello world");

// 接受第一页的吸收器
c.Pages[1].Accept(absorber);

// 改变第一个文本出现的第一个文本段的前景色
sorber.TextFragments[1].Segments[1].TextState.ForegroundColor = Color.FromRgb(System.Drawing.Color.Red);
// 更改第一个文本出现的第一个文本段的字体大小
sorber.TextFragments[1].Segments[1].TextState.FontSize = 15;

// 保存文档
c.Save(@"D:\Tests\output.pdf");  
```
